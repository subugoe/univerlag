Veröffentlichung einer Publikation Schritt für Schritt

Arbeitsschritte, die außerhalb des Systems abgewickelt werden, sind grau markiert.

Ist ein Dokument zur Publikation eingereicht, so liegt es im Bearbeitungspool und die Erledigung des nächsten Schrittes im Workflow kann von einem der Univerlag-Mitarbeitern übernommen werden. Der anstehende Schritt ist im Bearbeitungspool als Aufgabe angegeben.

Die Veröffentlichung einer Publikation in den Sammlungen des Verlagsprogramms ist zweistufig.

1. Schritt: Grundesätzliche Entscheidung, ob ein eingereichtes Dokument veröffentlichungswürdig ist. (Aufgabe: "Begutachtung")

  • Das eingereichte Dokument herunterladen:
    im Bearbeitungspool auf den Titel des Dokumentes klicken
    => Das leere Coverfeld anklicken.
  • Entscheiden ob das Dokument alle Voraussetzungen zur Veröffentlichung bei Univerlag erfüllt.
  • Aufgabe "Wartet auf Begutachtung" aus der Bearbeitungspool übernehmen
    Je nach Entscheidung die Publikation annehmen oder ablehnen.

Im Falle einer Ablehnung erscheint ein Textfeld. Hier können Sie dem Autor des Dokumentes mitteilen, warum die Publikation abgelehnt wird. Dieser Text wird an den Autor als E-Mail-Nachricht geschickt.

Wird das Dokument angenommen, so landet es wieder im Bearbeitungspool und wartet auf den zweiten Bearbeitungsschritt.

Preis und alle Modalitäten werden mit dem Autor geklärt, das Dokument durchläuft einen Peer-Review Prozess - falls vorgesehen - und wird vom Verlag zur Publikation bearbeitet.

2. Schritt: Publikation ablehnen oder Veröffentlichung eines fertigen Verlagsdokumentes (Aufgabe "Verlagsbearbeitung")

2. a) Das Dokument wird doch nicht veröffentlicht.

Publikation aus der Bearbeitungspool übernehmen
Publikation ablehnen

2. b) Das Dokument ist angenommen und bearbeitet und steht zur Veröffentlichung bereit.

Verlagsfertiges Dokument mit dem eingereichten Dokument austauschen und Coverbild hochladen
Zuerst muss die ursprüngliche Datei gelöscht, dann das fertige Verlagsdokument hochgeladen werden. Dazu braucht man die Publikation nicht aus dem Bearbeitungspool übernehmen!

Unter Hauptmenü "Veröffentlichungen" auf dem Titel der der Publikation klicken.
Über den leeren Cover auf den Link "Dokument bearbeiten" klicken.
=> den Reiter "Zugehörige Dateien" auswählen
=> Checkbox neben dem ursprünglichen Dokument einhacken => auf "Dateien löschen" klicken

Ebenfalls auf dieser Seite:
auf den Link "Eine neue Datei hochladen" klicken
=> Knopf "Datei auswählen" betätigen
= Verlagsdokument auf dem lokalen Rechner auswählen
=> den Knopf "Hochladen" drücken.

Um das Coverbild hochzuladen:

Den Link "Eine neue Datei hochladen" anklicken
=> im obersten Dropdown-Menü "Vorschaubilder" auswählen
=> "Datei auswählen" klicken
=> Coverbild auf dem lokalen Rechner auswählen
=> Knopf "Hochladen" betätigen.

Metadaten der Publikation ergänzen

Im Hauptmenü "Veröffentlichungen" die Publikation aus der Bearbeitungspool nehmen
Unter "Ihre eigenen Aufgaben" die Aufgabe "Verlagbearbeitung" neben dem Titel der Publikation anklicken
=> Knopf "Metadaten bearbeiten" drücken
=> einfach "Weiter"
=> die Metadaten, die vom Autor eingegeben wurden, durchlesen und ggf. Korrigieren
=> Unter der Zusammenfassung kommen die Felder, die nur für Editoren sichtbar sind. Alle relevanten Felder gemäß der Beschreibung ausfüllen. Siehe dazu Spezielle Metadatenfelder! => "Weiter" bis alle Schritte durchlaufen sind
=> "annehmen".
In einigen Minuten ist die neue Publikation auf der Startseite als Neuerscheinung zu finden!

Falls die neue Publikation in einer Reihe erschienen ist, sind noch folgende Schritte nötig:

ein neues Feld "dc.bibliographicCitation.volume" erzeugen und die Bandzählung mindestens zweistellig (mit führendem "0") eintragen. Dies ist für die Sortierung nach Band notwendig!

Zur Publikation navigieren
=> im Hauptmenü "Dokument bearbeiten" wählen.
=> Reiter "Metadaten" auswählen
=> im obigen Dropdown-Menü "dc.bibliographicCitation.volume auswählen
=> im Textfeld darunter die Bandnummer eintragen
=> "metadatum hinzufügen" anklicken.

Publikation in die entspr. Reihe spiegeln.

Hauptmenü "Veröffentlichungen" wählen

Den Handle der zu spiegelnden Publikation kopieren

Beschreibung der Medienformate

dc.format.medium
Das Feld gibt an, welche medienformate zu einer Publikation existieren. (Auch wenn sie für den Nutzer nicht verfügbar sind!) Das Feld arbeitet mit kontrolliertem Vokabular. Mögliche Werte sind: "Print", "DC-ROM" oder "DVD". Es soll mehrfach, d.h. für jeden Medientyp wiederholt werden. Achtung! Die Beschreibung eines Medientyps wird bei der Anzeige ignoriert, wenn kein dc.format.medium Feld mit dem entspr. Wert vorhanden ist.

Buchbeschreibung

Die Printversion einer Publikation wird mittels drei Metadatenfelder beschrieben:
  1. dc.description.print
    Beschreibung des Covers der Druckversion. Etwas "Softcover, 20x20".
  2. dc.format.extent
    Umfang des Buches: Seitenzahl (ohne "S.", "Seiten" o.ä.)
  3. dc.price.print
    Der Preis in €: Angegeben mit zwei Dezimalstellen, Komma als Dezimalpunkt und ohne Währungsnagabe
Diese Felder sind Pflicht, falls es eine Printversion existiert. dc.description.print

CD-ROM Beschreibung

Die CD-ROM-Version einer Publikation hat kein festes Plichtfeld. Welche Felder vorhanden sein müssen variiert je nachdem, ob sie zum Kauf angeboten wird oder nicht.
  1. CD ist kaufbar:
    dc.price.cdrom ist Pflicht.
    Der Preis in €: Angegeben mit zwei Dezimalstellen, Komma als Dezimalpunkt und ohne Währungsnagabe.
  2. CD ist nicht kaufbar.
    Dann muss das Feld dc.notes.cdromaccess mit einem der möglichen Werte angelegt werden: "outofstock", "notavailable" oder http:// zu der Onlineversion der CD.
Zusätzlich kann und ist empfohlen, eine Beschreibung der CD im Feld dc.description.cdrom anzugeben. Diese Beschreibung ist Freitext und kann eine Angabe zum Inhalt oder zum Spielzeit der CD sein. Die Spielzeit bitte in der Form "32:12 Min." angeben! Ist ein Feld dc.format.medium mit dem Inhalt "CD-ROM" vorhanden, so wird in der Anzeige automatisch die Beschriftung "CD-ROM:" für die vorhandenen Angaben generiert.

DVD Beschreibung

Die DVD-Version einer Publikation hat kein festes Plichtfeld. Welche Felder vorhanden sein müssen variiert je nachdem, ob sie zum Kauf angeboten wird oder nicht.
  1. CD ist kaufbar:
    dc.price.dvd ist Pflicht.
    Der Preis in €: Angegeben mit zwei Dezimalstellen, Komma als Dezimalpunkt und ohne Währungsnagabe.
  2. DVD ist nicht kaufbar.
    Dann muss das Feld dc.notes.dvdaccess mit einem der möglichen Werte angelegt werden: "outofstock", "notavailable" oder http:// zu der Onlineversion der DVD.
Zusätzlich kann und ist empfohlen, die Spielzeit der DVD im Feld dc.description.DVD anzugeben. Angabe als "45:04 Min."! Ist ein Feld dc.format.medium mit dem Inhalt "DVD" vorhanden, so wird in der Anzeige automatisch die Beschriftung "DVD-Video:" für die vorhandenen Angaben generiert.

Spezielle Metadatenfelder

  • dc.notes.access
    Normalerweise gehört zu jeder Publikation mindestens eine Printversion und ein Online-Dokument (PDF). Mit Hilfe dieses Feldes kann und sollte ein Hinweis angezeigt werden, daß einer dieser gängigen Formate nicht vorhanden ist.
  • dc.relation.otherparts
    wird nur bei mehrteiligen Publikationen benutzt. Es enthält den Verweis auf andere Teile der Publikation, die ebenfalls bei Univerlag erschienen sind und muss die folgende Syntax haben:
    Teilbezeichnung (z.B. "Teil 2" oder "Band 4": handle /<handle> => Anzeige: Text vor dem Doppelpunkt + Link auf die Publikation mit dem Handle
    Die Teilbezeichnung soll in der Sprache des Titels angeben werden! Beislpiel
  • dc.description.other
    Verknüpfung mit Online-Ressourcen außerhalb von Univerlag.
    Falls vorhanden, wird der Feldinhalt über dem Erscheinungsjahr angezeigt.
    Soll ein HTTP Link enthalten. => Anzeige: Link mit Überschrift "Online-Version"
    Beispiel
  • dc.relation.isreferencedby
    Verweise auf Rezensionen
    Kann einen HTTP Link auf eine Online-Rezension oder die bibliographische Angabe einer Druckrezension enthalten => Anzeige: Im Reiter "Rezensionen" je nach Angabe verlinkt oder nicht.
    Für mehrere Rezensionen bitte das Feld wiederholen! Beispiel
  • dc.bibliographicCitation
    Das Feld ermöglicht die Sortierung von Reihen nach Bandnummer.
    Der Feldinhalt ist nur in der Langanzeige sichtbar.
    Bitte nur die Bandnummer der Reihe mindestens zweistelling (mit führendem Null) eintragen! Falls die Numerierung Komma oder Punkt enthält, diese einfach weglassen. Slashes werden jedoch übernommen. z.B. bei SPE; 1,1 => 11 eingeben
    Annals of the History and Philosophy of Biology 16/2011 => 16/2011
    InterKultur; 1 => 01
  • dc.notes.intern
    Interne Bemerkungen, die für normale Nutzer nicht sichtbar sind.
    Dies kann alles Mögliche sein: wie Verhandlungsdetails mit dem Autor zur Publikation, erzwungene Änderungen, ToDos etc.

Mehrteilige Publikationen

Jedes Teil erscheint als eigenständige Publikation mit der Teilangabe im Titel in derselben Sprache. Die einzelnen Teile werden über das Feld dc.relation.otherparts miteinander verknüpft. Siehe dazu hier!

Bereits erschienene Publikationen (Metadaten oder Dokumente) bearbeiten

  1. Zum Dokument navigieren:
    • Hauptmenü rechts/Kontext: Dokument bearbeiten oder
    • Hauptmenü rechts/Administration: Dokumente => Handle angeben
  2. Metadaten bearbeiten: über Reiter “Metadaten”
  3. Dokumente bearbeiten: über Reiter “Zugehörige Dokumente”

Neuerscheinungsliste kontrollieren

Welche Publikationen als Neuerscheinungen aufgelistet werden, hängt von dem Feld dc.date.accessioned = Einstellungsdatum ab. Sie listet die zuletzt freigegeben Publikationen auf. Soll eine Publikation vor den anderen in der Neuerscheinungsliste angezeigt werden oder eine Publikation in die Liste aufgenommen werden, muss dieses Datum entspr. geändert werden.

Reihenfolge der Dokumente innerhalb Publikationen ändern

Dokument bearbeiten => Reiter "Zugehörige Dateien" => mit den Pfeilen die Reienfolge ändern => Button "Reihenfolge ändern"!

Mögliche Zustände einer Publikation

  • verworfen: Die Publikation ist aus dem Repository zurückgezogen. Sie ist aus der Suche/Browsing herausgenommen. Die Persistente URL wird aber gepflegt. Bei Aufruf via Handle erscheint die Meldung, daß diese Publikation zurückgezogen wurde.

    Dieser Zustand kann wieder rückgängig gemacht werden. Beispiel für Nutzungsszenario: Publikation muss wegen Plagiat zurückgenommen werden.
  • nicht sichtbar: Die Publikation ist ist aus der Suche/Browsing herausgenommen. Jedoch über ihr Handle direkt aufrufbar.
    Dieser Zustand kann wieder rückgängig gemacht werden.
    Beispiel für Nutzungsszenario: ??
  • nicht sichtbar: Die Publikation ist aus der Suche/Browsing herausgenommen. Jedoch über ihr Handle direkt aufrufbar.

    Beispiel für Nutzungsszenario: ??
  • endgültig gelöscht: Die Publikation ist aus dem System entfernt worden.
    Dieser Zustand kann nicht rückgängig gemacht werden!
    Beispiel für Nutzungsszenario: Bitte nie nutzen! Statt dessen immer Publikation verwerfen!

Zustände einer Publikation ändern

  1. Zum Dokument navigieren:
    • Hauptmenü rechts/Kontext: Dokument bearbeiten oder
    • Hauptmenü rechts/Administration: Dokumente => Handle angeben
  2. Im Reiter “Dokumentenstatus”
    den entsprechenden Knopf auswählen und Auswahl bestätigen.

Diese Methoden können nur den Status einer ganzen Publikation ändern. Die Sichtbarkeit einzelner Dateien kann man über das Embarogmanagement oder durch händische Änderung der Zugriffsrechte der Dateien kontrollieren!

Interne Identifizierung der Publikationen

via Handle: Teil der URL nach dem Wort handle
z.B. "56789/531" in http://univerlag.uni-goettingen.de/handle/56789/531

oder

Navigation im System

Suche oben rechts sucht standardmäßig im Verlagsprogramm. Um auch verlagsfremde Publikationen zu finden: Suche im starten => im Dropdown “Suche” “Gesamter Bereich” auswählen und noch einmal abschicken!

Suche in einer Sammlung: Zur Sammlung navigieren und in der Einstiegseite der Sammlung die Suche benutzen.

ISBN-Suche:: ISBN in Anfürhungszeichen im Suchfeld eingeben.

Allgemeiner Hinweis zur Suche:Identifikatioren (ISSN, PPN, Handle etc. immer in Anführungszeichen setzen!

Browsing: Mit einem Klick nur in in den einzelnen Bereichen möglich. In allen Bereichen blättern: auf einen bel. Bereich klicken => im Krümelpfad im Kopfbereich auf “Alle Produkte” klicken.
